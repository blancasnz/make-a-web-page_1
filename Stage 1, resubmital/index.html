Basics of the Web and HTML

The Web

The basics of the web include The Web, HTML, URLS, HTTP, and Web Applications.

Major pieces of the web include You, the Browser, the Internet, HTTP, and Servers. When you go on a web browser (such as Chrome or Firefox), you connect to the Internet which connects to an HTTP (main protocol of the web) which then connects to a Server. The Server responds back with the correct page of what you searched for through HTTP which sends it back to your browser for you to view.

HTML

The worldwide web is made up of HTML documents. HTML stands for Hypertext Markup Language. It is the basis for everything on the web (what glues everything together). HTML is made up of text content (what you see), markup (what it looks like/arranged), and references to other documents (eg. images, videos, links to other pages).

HTML TAGS: b tag (bold), em tag (emphasis in italic), a tag (anchor tag used for making links by creating attributes), img tag (images), br tag (line breaks), and p tag (paragraphs).

Tags are either block elements (start on a new line/invisible box around them) or inline elements (just end a line/no invisible box around them).

Block elements include - p, div, form
Inline elements include - a, span, br, img, strong

Most tags have an opening tag and a closing tag (ex. p tag). Tags that don't require closing tags are called "void tags" (ex. br tag).

Main Points

What is a web page?
A web page is a text document written in language called HTML. Web browsers read these documents, and then interpret and display them.

How coding works
When programmers write text in a language that a computer can understand, the computer can then follow the instructions the programmer wrote.

Computers are stupid
Programmers need to write exactly the way a computer understands (also known as writing the correct "syntax").

Programmers Can't Remember Everything!

HTML tags are always contained within angled brackets. HTML elements refer to everything within a set of opening and closing tags. An attribute is a property of an HTML element (href attribute of anchor tag).

HTML and CSS Structure

HTML and CSS develop and change over time but the underline principles stay the same.

Website = House
HTML = Walls
CSS = Color of the walls/carpet
Javascript = Interactive components/changing an element of the house.

Tree Like Structures

You can look at the structure of a page by going into the Developer Tools. A tree like/branching structure appears and by expanding the element, one can go deeper and deeper into the structure and view how it all connects, just like a family tree. When programmers are talking about the DOM (Document Object Model), they are referring to the tree-like structure of the page. It is important to have structure and organization because it allows the programmer to easily edit or make changes.

Everything on a webpage is a box!

Text Editors

A text editor is a program that allows you to edit your text. Scratchpad and Codepen are beginner friendly text editors. Sublime is a more advanced text editor that has a wide range of features such as supporting various prgramming languages.

Adding CSS style to HTML Structure

CSS

CSS stands for Cascading Style-Sheets. Cascading means that rules are applied not only to the elements they directly match, but also to all of those elements' child elements. However, if a child element has multiple, overlapping rules defined for it, the more specific rule takes effect.

Inheritance

Inheritance is a key feature in CSS. It relies on ancestor-descendant relationships to operate. Properties are applied not only to specified element, but also to it's descendants. Inheritance prevents certain properties from being declared over and over again, allowing less CSS to be written and to also have a faster web page loading.

Don't Repeat Yourself

There are often changes and modifications that programmers make to their code. Avoiding repetition in coding makes the process much simpler by allowing the programmer to make changes in one place instead of many. When there are many lines of the same code, a programmer has to make changes to all of it and problems/errors can arise if all changes aren't made. Avoiding repetition makes life easier!
